GitHub SSH Key 全流程（Windows + WSL）
适用于：
• 新电脑
• 私有仓库 clone
• Windows + WSL 双环境
• 长期科研开发环境
 
一、为什么使用 SSH
GitHub 已经禁止密码登录，必须使用：
• SSH key（推荐）
• Personal Access Token
SSH 优点：
• 永久可用
• 不需要每次输入密码
• 支持私有仓库
• 多仓库最稳定
科研开发推荐 SSH
 
二、Windows 主系统生成 SSH Key
打开 PowerShell
Bash
Copy
 
ssh-keygen -t ed25519 -C "你的github邮箱"
例如：
Bash
Copy
 
ssh-keygen -t ed25519 -C "wmdmatt@gmail.com"
系统提示：
Bash
Copy
 
Enter file in which to save the key
直接按回车
继续提示：
Bash
Copy
 
Enter passphrase
直接回车
再次回车确认
 
三、查看 SSH Public Key
PowerShell 输入
Bash
Copy
 
cat ~/.ssh/id_ed25519.pub
如果报错使用
Bash
Copy
 
type C:\Users\Administrator\.ssh\id_ed25519.pub
输出类似：
Copy
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... wmdmatt@gmail.com
复制整行
 
四、添加 SSH Key 到 GitHub
打开：
https://github.com/settings/keys
点击：
New SSH key
填写：
Title
例如：
Copy
Windows Laptop
Key
粘贴刚复制的 key
点击：
Add SSH key
 
五、测试 SSH 连接
Bash
Copy
 
ssh -T git@github.com
第一次可能提示：
Bash
Copy
 
Are you sure you want to continue connecting
输入：
Bash
Copy
 
yes
成功输出：
Bash
Copy
 
Hi CrazyRunningRabbit!
You've successfully authenticated
说明成功
 
六、克隆私有仓库
进入工作目录
Bash
Copy
 
cd C:\
mkdir research
cd research
克隆仓库
Bash
Copy
 
git clone git@github.com:CrazyRunningRabbit/仓库名.git
例如：
Bash
Copy
 
git clone git@github.com:CrazyRunningRabbit/HaselTO_packedpara.git
 
七、WSL 需要单独配置 SSH
Windows 和 WSL 是两个系统
必须分别配置 SSH
进入 WSL
Bash
Copy
 
wsl
生成 SSH key
Bash
Copy
 
ssh-keygen -t ed25519 -C "wmdmatt@gmail.com"
查看 key
Bash
Copy
 
cat ~/.ssh/id_ed25519.pub
复制
 
八、添加 WSL SSH Key 到 GitHub
再次打开
https://github.com/settings/keys
点击：
New SSH key
Title：
Copy
WSL Ubuntu
粘贴 key
保存
 
九、测试 WSL SSH
Bash
Copy
 
ssh -T git@github.com
成功输出：
Bash
Copy

③ git clone

十二、常见问题
1. Permission denied
说明 SSH 未配置
重新执行
Bash
Copy
 
ssh-keygen
 
2. Clone 失败
检查
Bash
Copy
 
ssh -T git@github.com
 
3. clone 空仓库
检查分支
Bash
Copy
 
git branch -a
切换
Bash
Copy
 
git checkout master
